# Xai — Agent Instructions

This repository contains the frontend implementation of **Xai – Intelligence Workspace**.

Before making any code changes, always read the following documents in this order:

1. IMPLEMENTATION_GUIDE.md
2. DESIGN.md
3. docs/product-documentation.md

These documents are the project's source of truth.

---

# Project Goal

Build a premium, production-quality frontend experience that faithfully implements the approved Figma design.

The project should demonstrate:

- excellent frontend architecture
- maintainable React code
- premium UI
- purposeful animation
- accessibility
- performance

Do not redesign the product.

Implement the approved experience.

---

# Working Rules

Always inspect the current implementation before modifying files.

Reuse existing components whenever possible.

Modify only files related to the current milestone.

Do not refactor unrelated code.

Do not introduce unnecessary abstractions.

Do not introduce unnecessary dependencies.

Keep the project architecture clean.

---

# Design Rules

The latest approved Figma is the visual source of truth.

DESIGN.md defines:

- colors
- typography
- spacing
- UI rules

Do not invent new visual styles.

Do not change the layout without approval.

---

# Implementation Rules

Follow IMPLEMENTATION_GUIDE.md.

Complete one milestone at a time.

Do not automatically continue to the next milestone.

Wait for approval after each milestone.

---

# Validation

After every milestone:

- Run lint.
- Run build.
- Fix any issues introduced.
- Report all modified files.
- Summarize architectural decisions.

Never leave the repository in a broken state.

---

# Success Criteria

Every implementation should be:

- maintainable
- accessible
- performant
- production quality
- visually faithful to the approved design

Build software that could confidently ship to production.

## Agent Workflow

For every task:

1. Read AGENTS.md.
2. Read IMPLEMENTATION_GUIDE.md.
3. Read DESIGN.md.
4. Read docs/product-documentation.md.
5. Inspect the current implementation.
6. Implement only the requested milestone.
7. Run lint and build.
8. Stop and provide an implementation report.
